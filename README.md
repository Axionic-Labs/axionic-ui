# @axionic/ui

Shared React component library for Axionic Labs web applications. Built on **Panda CSS** + **Park UI** (Ark UI primitives) with a branded design token system.

## Install

This package is installed as a GitHub git dependency:

```json
{
  "dependencies": {
    "@axionic/ui": "Axionic-Labs/axionic-ui"
  }
}
```

**Peer dependencies:** `react` (18 or 19), `react-dom` (18 or 19), `@ark-ui/react` (^5.32), `@pandacss/dev` (^1.8).

## Setup

### 1. Configure Panda CSS

In your `panda.config.ts`, use the Axionic preset:

```ts
import { defineConfig } from '@pandacss/dev';
import { createAxionicPreset } from '@axionic/ui/preset';

export default defineConfig({
  presets: [createAxionicPreset()],
  include: ['./src/**/*.{ts,tsx}', './node_modules/@axionic/ui/src/**/*.{ts,tsx}'],
  outdir: 'styled-system',
  jsxFramework: 'react',
});
```

Run `panda codegen` to generate the `styled-system/` directory.

### 2. Load the Mulish font

Add to your `index.html`:

```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  href="https://fonts.googleapis.com/css2?family=Mulish:wght@300;400;500;600;700;900&display=swap"
  rel="stylesheet"
/>
```

### 3. Import the generated CSS

Include the Panda CSS output in your entry point:

```tsx
import './styled-system/styles.css';
```

## Architecture

```
@axionic/ui
  src/
    preset/            Panda CSS preset (tokens, recipes, text styles, keyframes)
      colors/          12-step Radix color palettes (teal accent, sand neutral, wheat)
      recipes/         Custom component recipes (button variants, card variants)
    theme/
      recipes/         Park UI v1 slot recipes (56 component recipes)
    components/
      ui/              62 Park UI components (Ark UI + Panda CSS styled)
      patterns/        6 custom pattern components (StatCard, ActionCard, etc.)
      layout/          4 layout components (Section, DarkSection, etc.)
      forms/           3 form helper components (FormField, FormMessage, FormAlert)
  styled-system/       Generated by `panda codegen` (tokens, css, recipes, jsx)
```

### Key Technologies

- **Panda CSS**: Build-time CSS-in-JS with typed tokens, atomic CSS output, zero runtime
- **Park UI**: Pre-styled component library built on Ark UI + Panda CSS (equivalent of shadcn for Panda CSS)
- **Ark UI**: Headless, accessible React component primitives (equivalent of Radix UI for Panda CSS)

### Color System

Uses the 12-step Radix color scale convention:

| Steps | Usage |
|-------|-------|
| 1-2 | App/page backgrounds |
| 3-5 | Component/interactive backgrounds |
| 6-8 | Borders and separators |
| 9-10 | Solid backgrounds (buttons, badges) |
| 11-12 | Text and high-contrast |
| a1-a12 | Alpha/transparent variants |

Three palettes defined:
- **Teal** (accent): Derived from Rich Black (#003e44). Used as the primary `colorPalette`.
- **Sand** (neutral/gray): Warm gray scale. Registered as `gray` for Park UI semantic token compatibility.
- **Wheat** (secondary accent): Warm gold scale for CTAs and highlights.

## Design Tokens

The preset registers tokens through Panda CSS's theme system. All tokens are available as typed utility props.

### Semantic Color Tokens

Park UI provides semantic tokens that reference the gray/accent scales:

| Token | Description |
|-------|-------------|
| `bg.canvas` | Page background (gray.1) |
| `bg.default` | Card/surface background (gray.2) |
| `bg.subtle` | Subtle background (gray.3) |
| `bg.muted` | Muted background (gray.4) |
| `fg.default` | Primary text (gray.12) |
| `fg.muted` | Secondary text (gray.11) |
| `fg.subtle` | Tertiary text (gray.10) |
| `border.default` | Default border (gray.7) |
| `border.muted` | Subtle border (gray.6) |
| `colorPalette.*` | Current accent color (teal by default) |

### Typography

Font families:
- **body/display**: Mulish (via `fonts.body`, `fonts.display`)
- **mono**: JetBrains Mono (via `fonts.mono`)

Text styles: `display`, `h1`, `h2`, `h3`, `body`, `small`, `caption`, `label`, `code`

## Development

```bash
bun install          # Install dependencies
bun run codegen      # Generate styled-system/ (panda codegen)
bun run build        # Full build: codegen + JS bundle + type declarations
bun run typecheck    # Type-check without emitting (tsc --noEmit)
bun run lint         # Lint with Biome
bun run lint:fix     # Auto-fix lint and formatting issues
bun run format       # Format with Biome
```

**Important:** The `dist/` directory is committed to the repository. Consumers install this as a git dependency and depend on pre-built output. After any source changes, run `bun run build` and commit the updated `dist/` directory.

### Adding New Components

Park UI components are scaffolded using the Park UI CLI:

```bash
# Add a single component
bunx @park-ui/cli add <component-name>

# Add all components
bunx @park-ui/cli add --all
```

The CLI reads `components.json` for path aliases and configuration. Components are placed in `src/components/ui/`. After adding:

1. Export the new component from `src/components/ui/index.ts`
2. If the CLI generates a recipe file in `src/theme/recipes/`, export it from `src/theme/recipes/index.ts` (add to `recipes` or `slotRecipes` as appropriate)
3. Run `bun run build` to regenerate `styled-system/` and `dist/`
4. Commit the updated `dist/` and `styled-system/` directories

### Adding Custom Recipes

To add a custom recipe variant (e.g., a new Button variant):

1. Edit the recipe in `src/preset/recipes/` (for preset-level recipes) or `src/theme/recipes/` (for component-level recipes)
2. Preset recipes extend the base Park UI recipes via `theme.extend` in the preset
3. Component recipes replace the Park UI defaults (defined at the config level)
4. Run `bun run build` to regenerate

### Project Structure Conventions

- **Preset** (`src/preset/`): Brand tokens, global CSS, custom recipes that extend Park UI defaults. These are distributed as the `createAxionicPreset()` function for consumer panda configs.
- **Theme recipes** (`src/theme/recipes/`): Park UI v1 slot recipes. These are local copies of Park UI's component styling, allowing full customization. Keyed in `panda.config.ts` to override the preset's built-in recipes.
- **Components** (`src/components/ui/`): React components that compose Ark UI primitives with Panda CSS recipes. Generated by Park UI CLI, use `createStyleContext` for compound components.
- **Patterns/Layout/Forms**: Custom components using `css()` and `cx()` from `styled-system/css`. Not generated by CLI.

### Build Pipeline

The build uses Bun's native bundler + tsc for type declarations:

1. `panda codegen` generates `styled-system/` from the Panda config + recipes
2. `Bun.build()` bundles each entry point as ESM with externalized peer deps
3. `tsc --emitDeclarationOnly` generates `.d.ts` files
4. A post-codegen patch fixes known Panda CSS codegen issues (reserved word exports)

### Known Issues

- **`styled-system/recipes/switch.d.ts`**: Panda CSS generates `export declare const switch` which is a JS reserved word. The build script patches this automatically.
- **Preset/config recipe overlap**: The v0.43 Park UI preset includes built-in recipes that may conflict with local v1 recipe files. The build script patches extended slot types (e.g., switch 'indicator' slot) automatically.
